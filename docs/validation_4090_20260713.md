# RTX 4090 CUPTI 验证报告

日期：2026-07-13

## 结论

本次验证证明 `overlap-monitor` 的原生 CUPTI 路径可以在 RTX 4090 上完成：

```text
真实 CUDA/NCCL kernel 采集
-> external correlation 关联
-> JSONL 导入与分类
-> NCCL/GEMM timeline overlap 计算
```

这属于功能性 smoke test，不是性能基准。测试时两张 GPU 均有其他任务，且
服务器未安装 TE 2.7.0 和 Nsight Systems，因此不能据此声明 profiler 开销、
Nsight 精度或真实 Megatron 1F1B 性能已经达标。

规格不同，本报告只记录实测输出，不据此推断硬件改装或虚拟化方式。

## 已测试

### 1. 离线逻辑和 CLI

| 测试 | 结果 |
| --- | --- |
| 单元/集成测试 | 31/31 通过 |
| synthetic CUPTI JSONL 导入 | 通过，2 events，0 dropped |
| critical-path synthetic 分析 | 通过，overlap ratio = 0.5625 |
| wheel 构建 | 通过，`overlap_monitor-0.3.0-py3-none-any.whl` |

### 2. 原生 collector 编译和加载

服务器镜像没有完整 CUDA Toolkit/nvcc，因此使用镜像已有的 CUDA、Triton
和 CUPTI headers 由 `g++` 构建共享库。结果：

```text
liboverlap_cupti.so 构建成功
libcupti.so.12 动态链接成功
start/stop 成功
push/pop external ID 成功
empty trace dropped_records = 0
```

### 3. 单卡 GEMM 采集

在 GPU0 上运行 10 次 FP16 `512 x 512` matrix multiplication：

| 指标 | 结果 |
| --- | ---: |
| CUPTI kernel records | 10 |
| dropped records | 0 |
| 单 kernel 时间 | 5.376-6.112 us |
| compute interval union | 55.25 us |
| 分类 | 10/10 为 GEMM |

### 4. 双卡 NCCL All-to-All + GEMM

可复现入口：

```bash
python3 -m torch.distributed.run --standalone --nproc-per-node=2 \
  -m overlap_monitor.benchmarks.cupti_gpu_smoke \
  --library build/cupti/liboverlap_cupti.so \
  --trace-dir traces/a2a \
  --iterations 5
```

每次 iteration 同时提交异步 `all_to_all_single` 和独立 stream GEMM。

| 指标 | rank0 | rank1 |
| --- | ---: | ---: |
| NCCL kernels | 5 | 5 |
| GEMM kernels | 5 | 5 |
| dropped records | 0 | 0 |
| externally linked kernels | 10/10 | 10/10 |
| communication union | 219.75 us | 413.50 us |
| compute union | 39.00 us | 44.75 us |
| actual intersection | 31.50 us | 38.75 us |
| overlap / min(comm, compute) | 0.8077 | 0.8659 |

GPU 上已有其他高利用率任务，因此 rank 间时间差和绝对耗时不可视为稳定性能
数据。本测试只验证采集、关联、分类和区间算法端到端可运行。

### 5. CPU 记录开销

服务器容器内 20,000 次、5 次取中位数：

| 指标 | 结果 |
| --- | ---: |
| bare fake `Work.wait()` | 30.75 ns/call |
| recorder wrap + wait | 3,032.36 ns/call |
| recorder 增量 | 3.00 us/call |
| 10,000 events 离线分析 | 12.09 ms |

该数据只覆盖 CPU Work recorder，不包含 CUPTI 对训练吞吐的影响。

## 测试中发现的问题

1. Docker 默认 `/dev/shm` 导致 NCCL 初始化失败；增加
   `--shm-size=1g` 后双卡测试通过。
2. 现有镜像没有完整 CUDA Toolkit，标准 CMake 路径未验证；本次使用镜像内
   pip CUDA/Triton headers 直接编译。
3. smoke trace 的 bubble ratio 没有 1F1B 调度语义，不能用于训练结论。

栈。
